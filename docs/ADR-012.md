# ADR-012: Phase 10.1 — The Material Foundation (Design Token Governance)

**Status:** ACCEPTED  
**Date:** 2026-05-11  
**Supersedes:** None  
**Related:** ADR-009 (Phase 9 — Advanced Authoring Station), ADR_PHASE_6_INDUSTRIALIZATION (Diagnostic System)

## Context

After the industrialization of the authoring logic in Phase 9, the OMEGA Manifest Editor required a similar hardening of its visual layer. Prior to Phase 10.1, aesthetic properties (colors, radii, materials, lighting) were scattered across multiple ad-hoc registries and local component overrides. This created "visual drift," where components rendered inconsistently across different tabs or document contexts, and made global skinning virtually impossible to manage at scale.

Furthermore, the `useDesignTokens.ts` hook had grown into a complex set of loosely typed utilities, leading to over 70 TypeScript errors during the initial attempt to centralize design constants. To achieve "Zero-Noise" production status, a formal governance model for design tokens was required.

## Decision

Phase 10.1 implements **The Material Foundation**, a centralized, immutable, and declarative design token system that decouples visual standards from component logic.

### 1. Immutable Token Registry (`design-tokens.ts`)

We establish `src/features/manifest-editor/constants/design-tokens.ts` as the single source of truth for the OMEGA visual identity.
- **Categorized Tokens**: Decouples values into logical buckets: `colors`, `radii`, `materials`, and `lighting`.
- **Industrial Standards**: Tokens use HSL-tailored palettes and specific industrial radii (e.g., `panel: '8px'`) to enforce the tech-noir aesthetic.
- **Semantic Immutability**: These tokens are constants that represent the "Factory Specs" of the OMEGA hardware.

### 2. Industrial Hybrid Hook (`useDesignTokens.ts`)

The `useDesignTokens` hook is refactored into a **Derivation Engine** that manages the hierarchy of visual values:
- **Merging Strategy**: The engine merges `DESIGN_TOKENS` (Factory Defaults) with `manifest.ui` overrides (User/Patch Specific) and local `entity` overrides.
- **Backward Compatibility**: To prevent breaking existing primitive renderers (`Knob`, `Port`, `Label`), the hook preserves legacy interfaces (`physics`, `resolveColor`, `resolveFont`) while deriving them from the new token registry.
- **Style Flattening**: The hook now flattens aesthetic variants directly onto the `style` object, allowing components to consume properties like `font` or `color` without nested property checks.

### 3. "Consumo Ciego" (Blind Consumption) Pattern

Components are transitioned to a **Blind Consumption** model via CSS Variables:
- The hook generates a `cssVars` map (e.g., `--omega-color-primary`, `--omega-rounding`).
- Core renderers (`CellNode`, `StructuralNode`) apply these variables to their root DOM elements.
- This allows CSS-level styling and animations to respond to token changes without re-rendering the React tree, improving performance in high-density rack views.

### 4. Zero-Noise Static Analysis

All `any` types in the aesthetic layer are replaced with strict `Record<string, unknown>` types or explicit interfaces like `DesignTokenOverrides` and `FlatStyle`. This resolves all `TS2339` and `TS7053` regressions and achieves full Era 7.2.3 compliance.

## Consequences

### Positive

- **Visual Consistency**: Every component across every document now shares the same material and lighting foundation.
- **Governance Scalability**: Global theme changes (e.g., changing the primary "Cyan" to a different shade) now happen in a single file (`design-tokens.ts`).
- **Performance**: Use of CSS variables reduces React prop-drilling for styles and enables hardware-accelerated transitions.
- **Developer Experience**: Zero TypeScript errors in the aesthetic pipeline ensures that future UI components are built on a stable, type-safe foundation.

### Negative

- **Initial Refactor Cost**: Widespread components needed update to import the hook from its new canonical path.
- **Mapping Complexity**: Maintaining an alias for `accent` (mapping to `primary`) adds a small layer of legacy debt to the hook to support existing patches.

## Validation Criteria

1. **Static Analysis**: `npx tsc --noEmit` and `npm run lint` must return zero errors. (Status: **PASS**)
2. **Visual Parity**: Injected blueprints must render with the correct tokens immediately upon insertion. (Status: **PASS**)
3. **Persistence**: Global aesthetic overrides in `manifest.ui` must survive document switching and hydration. (Status: **PASS**)
4. **Performance**: Viewport re-renders during slider movement must remain under 16ms (60fps) thanks to CSS variable offloading. (Status: **PASS**)
