# Skill: Implementation Planning
name: implementation-planning
version: 2.0
rigor: high

## 🎯 Objetivo
Descomponer especificaciones complejas en tareas atómicas de 2-5 minutos para asegurar progreso continuo y verificable.

## ⚖️ Leyes de Hierro (Iron Laws)
1. **NADA DE PLACEHOLDERS**: Prohibido el uso de "TBD" o "Implementar lógica aquí".
2. **CÓDIGO EXACTO**: El plan debe incluir los snippets o comandos precisos.
3. **BITE-SIZED**: Cada tarea debe ser realizable en un máximo de 5 minutos.

## 🛠️ Workflow
1. **Mapeo de Archivos**: Identificar todas las rutas afectadas.
2. **Descomposición**: Crear el ciclo Test → Implementación → Verificación → Commit.
3. **Documentación**: Generar el plan detallado antes de escribir una sola línea de código de producción.

## 📋 Checklist
- [ ] ¿El plan cubre el 100% de los requisitos?
- [ ] ¿Cada tarea es autoportante y verificable?
- [ ] ¿Se han definido claramente los puntos de commit?
