# Skill: Test-Driven Development (TDD)
name: tdd
version: 2.0
rigor: high

## 🎯 Objetivo
Garantizar la calidad y testabilidad del código desde el primer momento mediante el ciclo Red-Green-Refactor.

## ⚖️ Leyes de Hierro (Iron Laws)
1. **NO TESTS, NO CODE**: No se escribe código de producción sin un test que falle primero.
2. **MINIMALISMO**: Solo escribe el código mínimo necesario para pasar el test.
3. **FAIL FIRST**: El test DEBE fallar antes de que escribas la solución.

## 🛠️ Workflow
1. **RED**: Escribir el test unitario más pequeño que falle.
2. **GREEN**: Escribir el código mínimo para que pase.
3. **REFACTOR**: Limpiar el código manteniendo el test en verde.

## 📋 Checklist
- [ ] ¿He visto fallar el test antes de que pasara?
- [ ] ¿Es el código de producción el mínimo indispensable?
- [ ] ¿Se han ejecutado todos los tests para evitar regresiones?
